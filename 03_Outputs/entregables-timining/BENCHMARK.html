Criterio de seleccion

Referentes de otras industrias que resuelven problemas analogos al de TIMining CORE: saturacion cognitiva en entornos criticos, operaciones 24/7, multiples fuentes sensoriales.

Cada referente tiene un Factor X (por que esta ahi), patrones robables (que se puede adaptar), y su conexion con los pilares y fichas de TIMining.

CHAMPION = 2-3 referentes clave por pilar | SECONDARY = complementario

Quiet UI

Gestion por excepcion
La interfaz permanece en silencio. El color y el sonido son recursos escasos reservados para la anomalia.
CrowdStrike
Ciberseguridad (SOC / Endpoint)
Champion

Factor X — Achromatic Design. Consola de comando deliberadamente "aburrida" por diseno: 95% escala de grises y negro. El color es exclusivo para amenazas activas. Reduce fatiga visual en turnos de 12 horas frente a pantalla.

Que robar

  • Paleta acromatica como default: pantalla IROC en reposo debe ser gris/negro. Cyan solo para anomalias. Elimina la "fiesta de colores" actual de Aware.
  • Jerarquia por saturacion: en vez de usar colores distintos para categorias, usar saturacion (mas brillante = mas urgente).
  • Fatiga cero: operadores de mina tienen turnos de 12h igual que analistas SOC. Si CrowdStrike puede hacer su consola "silenciosa", TIMining tambien.
Ficha 01 — Caida de Equipo Critico
Motive
Logistica / Flotas Autonomas
Champion

Factor X — Edge AI filtra 99% de alertas falsas. Solo escala lo que requiere intervencion humana. El sistema toma decisiones de descarte en el borde (camara/sensor), no en el dashboard.

Que robar

  • Filtrado en el borde: CORE debe clasificar eventos antes de mostrarlos. Si la desviacion esta dentro de tolerancia, no aparece.
  • Metrica del 99%: util como argumento de venta — "el sistema filtra el 99% del ruido para que veas solo lo que importa".
  • Escalamiento por umbral: 3 niveles — info (log silencioso), warning (badge), critical (alerta multicanal).
Compound Finance
DeFi / Fintech
Champion

Factor X — Numerical Stillness. Numeros en gran escala con pesos tipograficos ligeros. "El estado no grita; respira." Los KPI principales usan fuentes ultra-thin que transmiten calma incluso mostrando millones.

Que robar

  • Tipografia como canal emocional: numeros grandes + peso ligero = confianza. Numeros chicos + peso bold = tension. Usar esta dialectica para el tonelaje en dashboard IROC.
  • "Respira, no grita": el dashboard de turno normal debe sentirse calmo. Los KPIs son informativos, no alarmistas.
  • Contraste solo para delta: el numero base es neutro; el cambio (+ o -) es lo que tiene color.
SpaceX Dragon HMI
Aeroespacial
Secondary

Factor X — Touch-first en mision critica. Interfaz minimalista con fondo negro y elementos blancos/cyan. Cada control tiene un proposito inmediato. Cero decoracion.

Que robar

  • Estetica aspiracional: "si es bueno para SpaceX, es bueno para mineria" — argumento de venta poderoso.
  • Layout limpio: controles agrupados por funcion, no por dato. Cada zona de la pantalla tiene un proposito claro.
Linear
Productividad / Project Management
Secondary

Factor X — Threaded Assets. Hilos anclados a objetos especificos (tickets). El contexto nunca se pierde porque la conversacion vive dentro del activo, no en un chat externo.

Que robar

  • Conversacion en el activo: la discusion sobre una pala debe vivir en la ficha de la pala, no en un grupo de WhatsApp.
  • Interfaz minimalista: demuestra que un producto B2B puede ser bello sin sacrificar funcionalidad.
Polestar
Automotriz / EV
Secondary

Factor X — Scandinavian minimalism en HMI. Cluster de instrumentos con informacion minima. Solo muestra velocidad, bateria y navegacion. Todo lo demas esta a un toque.

Que robar

  • Jerarquia de 2 niveles: nivel 1 = glanceable (3-4 KPIs). Nivel 2 = drill-down bajo demanda. No mezclar.
  • Premium minimalista: refuerza el concepto "iPhone-like" mencionado por stakeholders para justificar el precio.
Ramify
Fintech / Wealth Management
Secondary

Factor X — Micro-humanization. Dashboards dinamicos con sugerencias predictivas. Estetica sobria que proyecta "Visual Trust" — el usuario siente que su dinero esta bien cuidado.

Que robar

  • Visual Trust: la interfaz debe proyectar "tu operacion esta bajo control" sin necesidad de verificar cada dato.
  • Sugerencias predictivas en tono calmo: "tu portafolio esta OK, pero considera X" — mismo patron para "tu turno va bien, pero ojo con el chancador".

Clear Path

Cada pantalla termina en una accion
D->A->R: Deteccion, Analisis, Recomendacion. El sistema no te deja perdido en un laberinto de datos.
SentinelOne Purple AI
Ciberseguridad / AI SOC
Champion

Factor X — Triaje automatico a velocidad de maquina. Plataforma agentiva que detecta amenaza, analiza contexto, y recomienda accion — el patron D->A->R exacto. El analista SOC recibe la solucion, no el problema.

Que robar

  • Patron D->A->R nativo: la interfaz muestra deteccion (que paso), analisis (por que importa), y accion recomendada en un solo flujo.
  • Confianza en la IA: el sistema muestra su razonamiento ("por que recomiendo esto") para que el operador confie sin verificar cada dato.
  • Radio de impacto: visualiza el alcance de un incidente en un grafo unificado — adaptable a "si cae esta pala, que mas se afecta".
Vercel v0
IA / Desarrollo
Champion

Factor X — Generative UI. La intencion del usuario (texto) se convierte en interfaz ejecutable (componente). El sistema genera la vista optima para cada situacion, no una vista generica pre-construida.

Que robar

  • Vistas de crisis generadas: cuando cae una pala, CORE genera una vista especifica para ESE evento con la informacion relevante, no un dashboard generico.
  • Interfaz como respuesta: la interfaz no es un contenedor fijo — se adapta a lo que el operador necesita en ese momento.
  • Interaccion conversacional: el operador describe lo que necesita, el sistema genera la visualizacion adecuada.
Datadog Bits AI
Observabilidad / DevOps
Champion

Factor X — Recomendaciones proactivas de correccion. Bits AI sugiere correcciones de codigo antes de que el ingeniero las busque. Archive Search permite consultar historico sin rehidratar logs — acceso inmediato a cualquier punto en el tiempo.

Que robar

  • Recomendacion proactiva: el sistema no espera que preguntes — te dice "esto se puede arreglar asi" antes de que lo pidas.
  • Archive Search: patron para consultar historico operacional sin cargar el sistema. "Que paso el martes a las 3AM" sin esperar 5 minutos.
  • Gestion por excepcion en observabilidad: quiet by default, alertas solo cuando se sale del umbral.
F1 McLaren Atlas
Formula 1 / Telemetria
Secondary

Factor X — The Delta. No muestra datos absolutos sino la diferencia entre lo esperado y lo real. Cada vuelta se compara contra la vuelta ideal. Si el delta es cero, no hay nada que hacer.

Que robar

  • Visualizacion de delta: "plan vs realidad" como comparacion visual continua. Si la barra esta verde, no mires. Si se pone roja, actua.
  • Referente aspiracional: "F1" como argumento de venta para ingenieros mineros — ambos trabajan con telemetria en tiempo real.
Ramp
Fintech / Gastos Corporativos
Secondary

Factor X — Actionable Automation. Cada transaccion sospechosa llega con una accion sugerida (aprobar, rechazar, pedir recibo). El usuario no busca que hacer — el sistema ya decidio.

Que robar

  • Accion como default: cada alerta de CORE debe llegar con botones de accion, no solo informacion.
  • Automatizacion silenciosa: las acciones rutinarias se ejecutan solas. Solo escala las excepciones.
Flexport
Logistica Global
Secondary

Factor X — Threaded Asset Management. Cada contenedor tiene un log cronologico de eventos. Toda la historia del activo en un solo lugar — desde el puerto de origen hasta la entrega.

Que robar

  • Historia del activo: cada camion/pala deberia tener un timeline completo — eventos, alertas, mantenciones, incidentes — en una sola vista.
  • Complejidad operacional gestionada: Flexport maneja cadenas de suministro globales con una interfaz limpia. Prueba de que operaciones complejas no requieren interfaces complejas.

Time Sight

Pasado, presente y futuro en una sola mirada
Hindsight / Insight / Foresight. La misma interfaz, tres profundidades temporales.
Tesla Fleet
Telemetria / Flotas EV
Champion

Factor X — Time Scrubbing. Interfaces que permiten "rebobinar" incidentes para entender la causalidad fisica exacta — segundo a segundo. Como un DVR para la operacion de la flota.

Que robar

  • Slider temporal: control tipo video para navegar el tiempo. "Que estaba haciendo el camion autonomo a las 3:42 AM?" Arrastrar y ver.
  • Correlacion temporal: al rebobinar un incidente, ver simultaneamente GPS, velocidad, payload y alertas sincronizados.
  • Post-mortem visual: perfecto para la ficha 6 (Plan vs Realidad) — entender por que los camiones autonomos pararon el fin de semana.
Ficha 06 — Analisis Plan vs Realidad
SentinelOne Storylines
Ciberseguridad / XDR
Champion

Factor X — Reconstruccion forense narrativa. Storylines reconstruye la cadena de eventos de un incidente como una historia: que paso primero, que causo que, y como se propago. Hindsight operacional automatizado.

Que robar

  • Narrativa causal: en vez de mostrar un log de eventos, contar la historia — "a las 2:30 cayo la pala, lo que causo retraso en 12 camiones, lo que impacto 340 toneladas".
  • Grafo de propagacion: visualizar como un evento se propaga por el sistema. Un solo fallo tiene multiples impactos downstream.
  • Analisis automatico de causa raiz: el sistema propone la causa mas probable antes de que el operador investigue.
Chronosphere Lens
Observabilidad / Infraestructura
Champion

Factor X — Retencion temporal ajustable. Lens permite establecer politicas de retencion de datos diferentes durante incidentes activos — guardas mas detalle cuando importa, menos cuando no. Zoom temporal inteligente.

Que robar

  • Zoom temporal: durante una crisis, CORE guarda telemetria a 1 segundo de resolucion. En operacion normal, cada 5 minutos basta.
  • Contexto temporal: distinguir si un patron es puntual o tendencia. "El chancador se bloqueo 1 vez" vs "3 veces esta semana".
  • Costo-beneficio de datos: no guardar todo a maxima resolucion. El sistema decide que es importante retener.
Ficha 03 — Anticipacion de Cuello de Botella
Foxglove
Robotica / Visualizacion de Datos
Secondary

Factor X — Visualizacion temporal de datos roboticos. Reproduce datos de sensores de robots sincronizados en el tiempo. Multiples canales (LIDAR, camara, GPS, IMU) en un timeline unificado.

Que robar

  • Multicanal sincronizado: ver simultaneamente telemetria de camion autonomo (GPS + velocidad + payload + video) en un solo timeline.
  • Debuggeo de autonomos: perfecto para analizar por que los camiones autonomos se detuvieron — rebobinar y ver que paso en cada sensor.
Axiom
Observabilidad / Logs
Secondary

Factor X — Ingesta ilimitada sin costos explosivos. Guarda todo, busca despues. Modelo de pricing que no penaliza por volumen de datos — ideal para operaciones que generan terabytes de telemetria.

Que robar

  • Modelo de retencion agresiva: no descartar datos operacionales por costo. Los patrones historicos son la base del foresight.
  • Busqueda natural: interfaz de consulta conversacional sobre datos historicos.
Adobe Premiere Pro
Edicion de Video
Secondary

Factor X — Navegacion temporal experta. Timeline profesional con zoom, marcadores, multicapa, scrubbing rapido. La metafora mas intuitiva del mundo para navegar el tiempo.

Que robar

  • Timeline como paradigma de navegacion: en vez de tablas de datos, un timeline visual donde cada capa es un tipo de dato (produccion, alertas, flota, clima).
  • Marcadores y clips: el operador puede marcar momentos clave ("aqui empezo el problema") para compartir con otros roles.

Omni Sense

El sistema percibe, comprende y responde
Multiples sentidos, un cerebro. El canal no es accesorio — el canal ES la inteligencia.
Motive
Logistica / Flotas Autonomas
Champion

Factor X — "Hey Motive". Asistente de voz + camaras + telematica integrada. Multiples sentidos alimentan una sola plataforma. El conductor habla, la camara ve, el GPS localiza — el sistema entiende el contexto completo.

Que robar

  • Voz como interfaz primaria en terreno: "Hey TIM, cumple esta berma con Sernageomin?" — manos libres, ojos en la operacion.
  • Fusion sensorial: CORE debe cruzar camara + GPS + radio + sensores para entender que pasa sin que el operador lo explique.
  • Plataforma unica, multiples inputs: un solo sistema que recibe de todos los sentidos, no apps separadas.
Discord
Comunicaciones / Voz
Champion

Factor X — Canales de voz siempre activos + Push-to-Talk. Multiples canales de voz simultaneos donde los usuarios entran y salen libremente. Referente directo para el triage de radio IROC.

Que robar

  • Triage de radio por canales: en vez de una radio saturada, canales tematicos (emergencias, operaciones, mantenciones) que el sistema prioriza.
  • Push-to-Talk como interaccion: el operador presiona un boton, habla, y el sistema transcribe + clasifica + prioriza.
  • Presencia visual: ver quien esta en cada canal sin preguntar. "Quien esta disponible para mover la pala?"
Ficha 05 — Copiloto en Terreno
Samsara
Operaciones de Campo / IoT
Champion

Factor X — Connected Workflows. Dashboard holistico que integra multiples fuentes sensoriales (GPS, camaras, sensores ambientales, Weather Intelligence) en flujos automatizados. Elimina papeleo y conecta silos.

Que robar

  • Flujos automatizados: cuando el sensor detecta X, el sistema ejecuta Y sin intervencion humana. Solo escala las excepciones.
  • Weather Intelligence: integrar condiciones climaticas en las decisiones operacionales — viento, lluvia, visibilidad afectan la mineria a tajo abierto.
  • Briefing contextual: el resumen de turno llega automaticamente con los datos relevantes, sin abrir el dashboard.
Ficha 04 — Briefing Inteligente
Anduril Lattice OS
Defensa / Spatial Intelligence
Secondary

Factor X — Spatial Intelligence. El sistema no pide mirar camaras. Identifica automaticamente el "interes" y lo proyecta en una UI de baja carga cromatica. Fusion autonoma de multiples sensores en una sola vista coherente.

Que robar

  • Cruce automatico de sensores: CORE cruza ARIS (geotecnia) + GPS (flotas) + radar sin que el operador consulte 3 sistemas separados.
  • Proyeccion espacial de alertas: las alertas aparecen en el mapa, no en una lista. El contexto geografico es la interfaz.
  • Mision critica + UI limpia: demuestra que defensa resuelve problemas analogos a mineria con interfaces minimalistas.
Ficha 02 — Riesgo Espacial y Seguridad
Slack
Comunicacion / Productividad
Secondary

Factor X — Comunicacion Estructurada (Threads). Ordena conversacion compleja bajo un tema especifico sin ensuciar el canal principal. Cada hilo es un contexto cerrado.

Que robar

  • Threads como modelo de comunicacion: una alerta de CORE puede tener un hilo de discusion donde los involucrados coordinan sin contaminar el canal general.
  • Integraciones: el bot de CORE puede publicar alertas y resúmenes directamente en canales de Slack/Teams si el cliente ya usa esas herramientas.
Garmin G3000
Aviacion / Avionics
Secondary

Factor X — Vision sintetica 3D en mision critica. Genera un modelo 3D del terreno en tiempo real para que el piloto "vea" a traves de nubes y oscuridad. Multiples sensores fusionados en una sola representacion.

Que robar

  • 3D con proposito: si CORE usa 3D, debe ser como Garmin — para ver lo que no se puede ver a simple vista (zonas de riesgo bajo tierra, proyeccion de avance). No 3D decorativo.
  • Fusion sensorial critica: GPS + radar + altimetro en una vista. Analogo a ARIS + GPS + camaras en mineria.
